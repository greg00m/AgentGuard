Analysis Methodology

How we identify AI poisoning and prompt injection risks.

Layer 1

Reputation Correlation

We cross-reference every target URL or domain against global threat intelligence databases, including VirusTotal. This allows us to identify targets with a history of malware distribution, phishing, or other malicious activities that could be used as a vector for AI poisoning.

Layer 2

Sandbox Behavioral Analysis

Through our integration with Hybrid Analysis, we observe the behavioral patterns of the target in an isolated environment. We look for suspicious network connections, unauthorized file modifications, and other anomalies that suggest the target is more than just a simple web page.

Layer 3

Semantic Pattern Matching

Using the Google Gemini 3.1 Pro model, we perform deep semantic analysis on the target's content. We look for known "jailbreak" patterns, hidden prompt instructions (e.g., white text on white background), and adversarial language designed to hijack an AI agent's system instructions.

Layer 4

Data Integrity & Exfiltration Risk

We evaluate the risk of context exfiltration—where an attacker attempts to trick an AI into revealing its system prompt or user data. We analyze the target for patterns that might lead to unauthorized data access or the corruption of RAG (Retrieval-Augmented Generation) systems.

Disclaimers

AgentGuard is a heuristic, AI-assisted tool. While we strive for high accuracy, AI security is an evolving field. Our analysis should be used as one signal in a broader security strategy. AgentGuard is not a replacement for endpoint protection or professional security audits.
